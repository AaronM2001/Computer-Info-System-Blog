7 Layers of Protocols:

Protocols- Sets of rules used by devices for communication. Protocols are rules devices follow in order to be compatible (set of standards). Each Protocol is developed in Layers with each Layer being responsible for different parts. OSI- Open Systems Interconnection model. TCP/IP- Transmission Control Protocol and Internet Protocol.

Layer OSI TCP/IP Data Unit Application Internet Protocol Suites Examples
7 Application Application Data Where Application and user communicate HTTP, DNS, DHCP, FTP. Telnet, SSH, SMTP, IMAP, NTP Provides functions to the user. Provides the service requested by the user.
6 Presentation Application Data Establishing or getting rid of connections between devices NTP, etc Converts different representations. Makes sure the data is usable and readable. Where encryption happens
5 Session Application Data Establishing or getting rid of connections between devices. NTP, etc… Manages task dialogue. Maintains connections. Responsible for controlling ports and sessions.
4 Transport Transport Segments Adds TCP/UDP to keep all the data and adds source port numbers. Issues with the server. Can’t get a response from the server. TCP, UDP Provides end-to-end delivery Transmit data using transmission protocols TCP UDP. Provides security at this layer.
3 Network Internet Packet IP address and routing. Destination IP addresses are added. Routers are here. IPv4, IPv6, ICMP, ICMPv6, etc… Sends packets over multiple links Decides which physical path the data will take.
2 Data Link Network Access Frames Physical addresses are added to the data. MAC addresses and Switches are here. MAC, ARP, Ethernet 802.3. CDP, LLDP, DSL, L2TP, Sends frames of information. Provides the format of data on the network.
1 Physical Network Access Bits Carries the data across physical hardware. SONET/SDH, etc… Sends bits as signals. Transmits raw bit stream over the physical hardware
